Exploring Raytracing

Trying to recreate raytracing without following tutorials.

First let's explain what raytracing is.
Raytracing is a rendering technique for generating an image by tracing the path of light and its interactions with the virtual scene's objects.
It is commonly used to generate highly realistic images or animations.
The main idea is to simulate the way light behaves in the real world, taking into account reflections, refractions, shadows, and other optical effects.

My first idea was to do it as physics works : rays coming from the light source. Then I realised that the number of rays would be so huge that no computer could handle it.
So I decided to do it the other way around : rays coming from the camera.
I had to think about the different mathematical formulas in order to redirect, intersect the rays with the objects, and calculate the color of the pixel.

The result is a simple raytracer that can render a scene with spheres, planes, lights and any combination of them.
As for my other projects I used Java to handle the algorithmic side of the problem and might try to optimise it in C++ later.
